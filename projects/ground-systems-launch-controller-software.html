Project

Ground Systems Launch Controller Software

Society for Advanced Rocket Propulsion (SARP)

Ground Systems Software Engineer

Built Python-based launch controller software that increased ground-system flexibility and mid-flight transmission modeling, while validating telemetry across Raspberry Pi, IGS, and OpenMCT for reliable mission operations.

Python Raspberry Pi Telemetry & testing Hardare-Software Debugging Unit testing

The Problem

The ground systems UI was outdated and difficult to read, making it hard for operators to monitor and control the rocket clearly during testing. Beyond readability, there was a critical operational gap: the system didn't support pulsing and actuating valves for both launch controllers from a single interface. If the team needed to switch between controllers, someone had to manually edit a configuration text file and restart the UI â€” a slow, error-prone step during time-sensitive operations. The software also lacked the transmission modeling and telemetry validation needed to fully trust what was being displayed.

UI snapshots

What I Did

  • Developed Python software for launch controllers, adding flexibility to ground systems and mid-flight transmission modeling.
  • Conducted hardware-software debugging directly on the controller, resolving switch activation logic and signal integrity issues.
  • Validated end-to-end telemetry through Raspberry Pi into IGS/OpenMCT, confirming accurate packet reception, parsing, and display.
  • Designed and executed unit tests for the launch controller codebase to harden key functionality under varied simulated conditions.
  • Authored in-depth documentation and led a two-person subgroup, mentoring peers and keeping delivery on track.

The Outcome

The updated UI is significantly more readable and exposes valve controls directly on the main page. Operators can now pulse and actuate valves for both controllers from a single interface, with no config file edits or restarts required. Telemetry was validated end-to-end through Raspberry Pi into IGS/OpenMCT, confirming accurate packet reception, parsing, and display. The codebase is backed by unit tests and in-depth documentation to support future contributors.
